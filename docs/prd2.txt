## 3Ô∏è‚É£ Create a global launcher script `orbit-mcp`

We‚Äôll create a tiny shell script that:

1. Activates the venv
2. Runs `server.py`
3. Uses env vars if needed

### Step 3.1 ‚Äì create a `bin` folder in your home

```bash
mkdir -p ~/bin
```

### Step 3.2 ‚Äì create `~/bin/orbit-mcp`

Open it in an editor (e.g. `nano ~/bin/orbit-mcp`) and put:

```bash
#!/usr/bin/env bash
set -e

# Path to your orbit-mcp project
ORBIT_DIR="$HOME/dev/orbit-mcp"

# Optional: global env vars for orbit
export ORBIT_ENV_CONFIG="$ORBIT_DIR/config/environments.yaml"
export ORBIT_LOG_LEVEL="INFO"

# Activate venv
source "$ORBIT_DIR/.venv/bin/activate"

# Run the MCP server
python "$ORBIT_DIR/server.py"
```

Then make it executable:

```bash
chmod +x ~/bin/orbit-mcp
```

Now from *any* directory you can test:

```bash
orbit-mcp
# (it should run and just sit there waiting; Ctrl+C to stop)
```

If that works, we‚Äôre good.

## 4Ô∏è‚É£ Add `~/bin` to your PATH (so any path can find it)

Add this line to your shell config (`~/.bashrc` or `~/.zshrc`):

```bash
export PATH="$HOME/bin:$PATH"
```

Then reload:

```bash
source ~/.bashrc     # or: source ~/.zshrc
```

Check:

```bash
which orbit-mcp
# should output: /home/ahmad/bin/orbit-mcp
```

At this point:

* `orbit-mcp` is callable from **any path** in your terminal.
* Env vars like `ORBIT_ENV_CONFIG` are set automatically by the script whenever it runs.

## 5Ô∏è‚É£ Wire this into Cursor MCP config

Now you can simplify your Cursor config to use the **command name**, not a full python path.

Create or edit:

```bash

```

Example content:

```jsonc

    "orbit-mcp": {
      "command": "orbit-mcp",
      "args": [],
      "env": {
        // You can override or add extra env here if needed:

      }
    }
  }

```

Notes:

* Cursor will now just run `orbit-mcp` like any other CLI.
* The script sets `ORBIT_ENV_CONFIG` and activates the venv, so you don‚Äôt have to care from Cursor side.
* The `env` in this JSON is *extra* or overrides; you can keep it minimal.

Restart Cursor, then in a chat say something like:

> ‚ÄúAsk the `orbit-mcp` server to call `ping` and show me the result.‚Äù

If the config + script are correct, you‚Äôll see `orbit-mcp` listed under MCP servers in Cursor and the tools (`ping`, `list_environments`, etc.) will show up.

## 6Ô∏è‚É£ Where to put env vars (summary)

You have 3 layers you can use:

1. **Inside the launcher script** (`~/bin/orbit-mcp`)

   * Good for orbit-specific settings:

     ```bash
     export ORBIT_ENV_CONFIG="$ORBIT_DIR/config/environments.yaml"
     export ORBIT_LOG_LEVEL="INFO"
     ```

2. **Shell global env** (in `~/.bashrc` / `~/.zshrc`)

   * For values you might want in other tools too:

     ```bash
     export ORBIT_ENV_CONFIG="$HOME/dev/orbit-mcp/config/environments.yaml"
     ```

3. **Cursor MCP config `env` block**

   * For Cursor-only overrides:

     ```jsonc
     "env": {
       "ORBIT_LOG_LEVEL": "DEBUG"
     }
     ```

In practice I‚Äôd do:

* Put `ORBIT_ENV_CONFIG` in the **script**,
* Use `ORBIT_LOG_LEVEL` override per client (Cursor) if you want more verbose logs.
